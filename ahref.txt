Ahrefs Reverse API - Documentation Technique


Date : 25/03/2026
Proxy : ah.sommaserver.click (proxy vers app.ahrefs.com)
Cible analysée : www.cointribune.com/


1. Architecture de l’API
2. Base URL : https://ah.sommaserver.click/v4/
3. Authentification : Cookie-based (cookie “proxy-session”)
4. Parametre obligatoire sur chaque requete : original-host=app.ahrefs.com
5. Content-Type des reponses : application/json
6. Methode principale : GET avec le payload JSON encode en URL dans le parametre “input”
7. Quelques endpoints POST avec body JSON (Content-Type: application/json)
2. Authentification
L’API utilise une authentification par cookie. Le cookie principal est “proxy-session”. Pour le recuperer 👍
1. Ouvrir le navigateur sur ah.sommaserver.click et se connecter
2. 2. Ouvrir les DevTools (F12) > Application > Cookies
3. 3. Copier la valeur du cookie “proxy-session”
Le cookie a une duree de vie limitee et devra etre renouvele regulierement.


Cookies identifies :
* proxy-session : cookie d’authentification principal (obligatoire)
* - intercom-session-dic5omcp : cookie Intercom (optionnel)
* - intercom-device-id-dic5omcp : cookie device Intercom (optionnel)
3. Structure JSON de base (parametre “input”)
La plupart des endpoints GET prennent un parametre “input” qui est un objet JSON encode en URL. Voici la structure de base pour analyser un domaine 👍
{
  “args”: {
    “mode”: “subdomains”,
    “protocol”: “both”,
    “url”: “DOMAINE_CIBLE/”,
    “multiTarget”: [“Single”, {
      “target”: “DOMAINE_CIBLE/”,
      “mode”: “subdomains”,
      “protocol”: “both”
    }],
    “compareDate”: null,
    “country”: “fr”,
    “backlinksFilter”: null,
    “best_links_filter”: “showAll”,
    “competitors”: []
  }
}


Parametres importants :
* mode : “subdomains” (inclut tous les sous-domaines), “prefix” (URL exacte + sous-pages), “domain” (domaine entier), “exact” (URL exacte)
* - country : code pays ISO (“fr”, “us”, “de”, etc.)
* - url / target : le domaine cible avec slash final

8. {

12.     “size”: 0,
13.     “offset”: 0,
14.     “order_by”: null,

16.     “timeout”: null
17.   },
18.   “chart”: {

Valeurs possibles pour timeFrame : “month1”, “month3”, “month6”, “year1”, “year2”, “all”

7. Template generique pour changer de domaine cible
Pour adapter a un autre domaine, utiliser ce script Python pour generer l’URL encodee 👍
import urllib.parse, json


DOMAINE = “example.com/”
COUNTRY = “fr”
ENDPOINT = “seGetDomainRating”


input_data = {
    “args”: {
        “mode”: “subdomains”,
        “protocol”: “both”,
        “url”: DOMAINE,
        “multiTarget”: [“Single”, {
            “target”: DOMAINE,
            “mode”: “subdomains”,
            “protocol”: “both”
        }],
        “compareDate”: None,
        “country”: COUNTRY,
        “backlinksFilter”: None,
        “best_links_filter”: “showAll”,
        “competitors”: []
    }
}


encoded = urllib.parse.quote(json.dumps(input_data))
url = f”https://ah.sommaserver.click/v4/{ENDPOINT}?input={encoded}&original-host=app.ahrefs.com”
print(f”curl -s ‘{url}’ -b ‘proxy-session=88c7KC42lQqq6xSl880sQ92741e39949SI6707953cxBd4j0K6n50WN222c00x3AUl5OXmvGzNYu546HfFG4mGFG89252n2Q9TwC1583’”)


8. Notes importantes
* Le cookie proxy-session expire regulierement. Prevoir un mecanisme de refresh automatique (se reconnecter au proxy et recuperer un nouveau cookie).
* - Toutes les reponses sont au format JSON, encapsulees dans un tableau [“Ok”, data] ou [“Error”, message].
* - Le parametre original-host=app.ahrefs.com est obligatoire sur CHAQUE requete (GET ou POST).
* - Les reponses sont cachees cote serveur (Cache-Control: private, max-age=900) soit 15 minutes.
* - Le header Vary indique : x-workspace-id, X-Client-Version, Accept-Encoding.
* - Le serveur tourne derriere Cloudflare.
* - Pour les POST, le Content-Type doit etre application/json et le body doit etre un objet JSON valide (pas null).
* - Les erreurs retournent : [“Error”, [“InvalidInput”, “message d’erreur”]].
* - Le domaine cible doit toujours se terminer par un slash (ex: “example.com/”).
* - Pour lire la reponse JSON proprement : ajouter | jq . a la fin du curl.
